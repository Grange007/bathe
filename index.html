Abstract

针对华清大学紫荆学生公寓 24:00 强制停水带来的极端 Deadline 挑战，本文提出了一套基于软硬件协同漏洞利用的极限洗浴调度框架。有别于传统的洗浴规划，我们首次将系统底层的残存热力池（Residual Thermal Pool, RTP）机制与计费终端时钟漂移模型相结合，提出了一种基于多节点热力旁路和时钟漂移的一种极限边缘洗浴行为调度策略（简称 和一为 策略）。

在英文语境下，该策略的全称为 A Marginal Showerology Scheduling Strategy Based on Multi-Terminal Thermal Bypass and Clock Drift，为便于学术交流，我们提取其核心拓扑特征缩写为 WTF 策略。实验表明，相较于 Baseline（放弃洗浴），WTF 策略能将零点后的有效洗浴时间显著延长，并在降低洗发水残留率方面展现出了卓越的鲁棒性。

Hardware Vulnerability & Temporal Modeling

为了实现极限状态下的成功洗浴，我们对紫荆公寓洗浴系统的底层客观环境进行了精确建模。研究发掘了两大核心漏洞：异步时钟漂移（Asynchronous Clock Drift） 与 残存热力池（Residual Thermal Pool, RTP）。

Figure 2. System Hardware Architecture. 华清大学紫荆学生公寓淋浴系统硬件架构与实验环境。(a) 无隔板三节点拓扑空间；(b) 顶置式末端流体输出节点；(c) 存在时钟漂移漏洞的异步计费与时序控制终端；(d) 核心温度调节枢纽与 RTP 释放阀。

1. 时钟漂移漏洞 (Clock Drift)

洗浴计费终端缺乏网络时间协议（NTP）同步。随着学期推进，内置晶振的误差逐渐累积，实际停水时间 $T_{cutoff}$ 可建模为：

Tcutoff(d) = Tddl + β · d + N(0, σ²)

实测表明，到学期中后段，该漏洞通常能提供长达 4 至 5 分钟的初始延迟容忍度（Delay Tolerance）。

Figure 3. Clock Drift Accumulation. 120 天周期内的计费机器时钟偏移实测散点图与线性拟合。这宝贵的几分钟构成了本策略的第一层“时间护城河”。

2. 残存热力池机制 (RTP)

当系统切断主水源后，通过将混水阀强制旋转至极冷档位，可触发物理旁路。冷水在喷出前，会将管路死区体积（Dead Zone Volume, $V_{dz}$）内残留的高温水排出。其水温依赖于上一位使用者的离开时间 $\tau_{idle}$，为后续洗浴提供了极其宝贵的 1.5 ~ 3.5 秒纯热水补偿。

The WTF Framework (MTRB Algorithm)

单节点的 RTP 储量显然不足以支撑全套洗浴。依托紫荆公寓单间内三个无遮挡喷头（Node $S_1, S_2, S_3$）的拓扑结构，我们提出了 多节点接力洗浴法 (Multi-Terminal Relay Bathing, MTRB)。其核心动作分解如下：

  • Phase 1: 节点 1 压榨。 在断水瞬间，极速开启 $S_1$ 冷水档，利用前 2 秒 RTP 热水冲洗头部。感受到断崖式降温后立即关闭。
  • Phase 2: 横向滑移 (Lateral Shift)。 [Safety Warning] 由于沾满沐浴露的瓷砖动摩擦因数 $\mu_k \approx 0.1$，个体必须采用低重心贴地拖行（Shuffle Walk）移动至 $S_2$。严禁双脚腾空跳跃，运动学安全方程需满足：$mv^2 / r \leq \mu_k mg$。
  • Phase 3: 节点 2 与 3 接力。 依次激活 $S_2$ 与 $S_3$，独占各自的 RTP 储量，分别完成躯干与下肢的清醒冲洗仪式。

Figure 4. Spatial Routing for MTRB. 单间三喷头空间拓扑与多节点接力洗浴法的滑移路径示意。洗浴者通过 S1 → S2 → S3 的次序依次释放各节点残余热水，并以低重心横向滑移控制侧滑风险。

Experimental Results

我们在极限真实环境下进行了实证评估。评价指标包含：有效抢救热水时长、泡沫残留率（Foam Residue Rate, FRR，越低越好）以及主观从容度（Subjective Composure Index, SCI，1-10分，越高越好）。如表 1 所示，WTF 框架在各项指标上均取得了显著优势。
